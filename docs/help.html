Fayara Settings Explained

This document explains the various settings available in the Fayara browser extension to customize your experience. Click on the sections to expand or collapse them.

Connection
Ollama URL:

This is the network address where your Ollama instance is running. Typically, if Ollama is running on your local machine, this will be http://127.0.0.1:11434 or http://localhost:11434. Ensure this URL is correct for Fayara to connect to your local AI models.

Chat
Temperature:

Controls the randomness of the AI's responses during chat. Lower values (e.g., 0.2) make the output more focused and deterministic, while higher values (e.g., 1.0) make it more creative and diverse. The default is usually around 0.7.

Max Tokens:

Sets the maximum number of tokens (words or parts of words) the AI can generate in a single chat response. This helps control the length of the AI's answers.

Stream Responses:

When checked, the AI's response will appear word-by-word (streamed) as it's being generated. When unchecked, you'll wait until the entire response is generated before it appears. Streaming provides faster initial feedback.

Proofread
Enable Proofread:

Check this box to enable the proofreading feature, which allows you to select text on a webpage and have an AI model check it for grammar and suggest improvements.

Proofread Model:

Select the specific Ollama model you want to use for the proofreading task. Some models might be better suited for grammar correction than others.

Temperature:

Controls the randomness for the proofreading model. Lower values are generally recommended for proofreading to ensure more accurate and consistent corrections.

Max Tokens:

Sets the maximum length of the suggested proofread text generated by the AI.

Autocomplete
Enable Autocomplete:

Check this box to enable the text autocomplete feature. As you type in text fields on webpages, Fayara will suggest text completions using an AI model.

Autocomplete Model:

Select the specific Ollama model you want to use for generating text suggestions.

Temperature:

Controls the randomness of the autocomplete suggestions. Higher values might offer more creative but potentially less relevant suggestions.

Max Generated Tokens:

Sets the maximum number of tokens the AI will generate for a single autocomplete suggestion.

Max Context Before Cursor (tokens):

Defines the maximum amount of text (in tokens) before your cursor that will be sent to the AI as context for generating suggestions.

Max Context After Cursor (tokens):

Defines the maximum amount of text (in tokens) after your cursor that will be sent to the AI as context. This is useful for models that support "fill-in-the-middle" tasks.

Auto Suggest Delay (ms):

The time (in milliseconds) Fayara waits after you stop typing before automatically triggering an autocomplete suggestion (if 'Auto Suggest' is enabled).

(Time to wait after typing)
Wait For Space:

If checked, automatic suggestions ('Auto Suggest') will only trigger after you press the spacebar, rather than just pausing.

(Only auto-suggest after spacebar)
Trigger Key:

Select the keyboard shortcut (Tab or Ctrl+Space) used to manually request an autocomplete suggestion or accept the current suggestion.

Auto Suggest:

If checked, Fayara will automatically try to generate suggestions after you pause typing (respecting the 'Auto Suggest Delay' and 'Wait For Space' settings). If unchecked, you must use the 'Trigger Key' to get suggestions.

(Suggest after pausing)
RAG & Embeddings

These settings configure how Fayara processes documents (like web pages or PDFs) for Retrieval-Augmented Generation (RAG), allowing the chat AI to use information from these documents as context.

Embedding Model:

Select the Ollama model used to create numerical representations (embeddings) of text chunks. These embeddings help find the most relevant parts of the document to answer your questions.

Chunk Size (tokens):

The maximum size (in tokens) of each text segment the document is broken into before creating embeddings. Larger chunks capture more context but might be less specific.

Overlap Size (tokens):

The number of tokens that will overlap between consecutive text chunks. Overlap helps ensure that context isn't lost at the boundaries between chunks.

Maximum Context Size (tokens):

The maximum total size (in tokens) of the relevant chunks retrieved from the document that will be added to the chat model's context when answering your questions based on the document.

Remember to click the "Save Settings" button in the extension panel after making any changes!
